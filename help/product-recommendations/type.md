# レコメンデーションタイプ

Adobe Commerceでは、サイトの様々なページにデプロイできる大量のレコメンデーションが提供されています。 すべてのレコメンデーションタイプはデータ主導型です。 これらは、行動データ、製品属性データ、指標に基づいて動作します。 簡単に参照できるように、レコメンデーションタイプは次のようにグループ化されています。

- [Personalized](#personalized)
- [クロスセルとアップセル](#crossup)
- [人気度](#popularity)
- [高パフォーマンス](#highperf)

ベストプラクティスとして、Adobeでは、レコメンデーションを使用する際に次のガイドラインに従うことをお勧めします。

- レコメンデーションタイプを多様化します。 同じ製品を何度も提案した場合、顧客はレコメンデーションを無視し始めます。

- 買い物かごページと注文確認ページに同じレコメンデーションをデプロイしないでください。 使用を検討する `Most Added to Cart` 買い物かごページと `Bought This, Bought That` 注文確認ページ用。

- サイトを整頓しておいてください。 同じページに 3 つを超えるレコメンデーションユニットをデプロイしないでください。

- あなたの店が服を売っている場合、 `More like this` レコメンデーションにより、表示されている製品の性別に一致しない性別固有の製品が示される場合があります。 このレコメンデーションタイプは、非衣料品カテゴリに対してのみ使用することを検討してください。

## Personalized {#personalized}

これらのレコメンデーションタイプは、サイト上の特定の買い物客の行動履歴に基づいて、製品をレコメンデーションします。

| タイプ | 説明 |
|---|---|
| お勧め | 各買い物客の現在および以前のオンサイトの行動に基づいて、製品をレコメンドします。 買い物客の閲覧と購入履歴に基づいて、関連性の高いレコメンデーションを表示します。 このレコメンデーションタイプは、ほとんどの買い物客がサイトでジャーニーを開始するホームページで有効です。 シグナルを生成していないサイトの初回買い物客がエクスペリエンスをパーソナライズする場合、Adobe Commerceでは最も多く閲覧されたレコメンデーションタイプに基づいて商品が表示されます。 しかし、買い物客がサイト上の製品とのやり取りを開始すると、推奨製品は行動にリアルタイムで調整されます。<br/><br/>**使用場所：**<br/> — ホームページ<br/> — カテゴリ&#x200B;<br/><br/>**推奨ラベル：**<br/>  — ちょうどあなたのために<br/> — お勧め<br/> — 買い物の流れに感動 |
| 最近表示された項目 | ブラウザーの履歴に基づいて、買い物客が最近閲覧した製品を表示します。 削除された商品は、レコメンデーションユニットによって削除されます。 ブラウザー履歴がない場合や、フィルタールールが適用される履歴が十分でない場合は、レコメンデーション単位は表示されません。 結果に含まれる製品が設定より少ない場合、レコメンデーション単位は返された製品のみを表示します。<br/><br/>**使用場所：**<br/> — ホームページ<br/> — カテゴリ<br/> — 製品の詳細<br/> — 買い物かご<br/> — 確認&#x200B;<br/><br/>**推奨ラベル：**<br/> — 最近表示された項目<br/> — もう一度見る |

## クロスセルとアップセル {#crossup}

これらのレコメンデーションタイプは、買い物客が好きなものを見つけやすくするために、ソーシャルプルーフで動作し、他の類似製品を見つけるのに役立ちます。

| タイプ | 説明 |
|---|---|
| これを閲覧し、それを閲覧した | 買い物客が現在閲覧している製品に対して、より頻繁に閲覧する製品をレコメンドします。<br/><br/>**使用場所：**<br/> — 製品の詳細<br/> — 買い物かご<br/> — 確認&#x200B;<br/><br/>**推奨ラベル：**<br/> — この製品を閲覧したお客様が (PDP) も閲覧した。 |
| これを見て、購入したもの | 現在の製品を表示した後、買い物客が過剰に買い物をする傾向がある製品を推奨します。 買い物客が気付かなかった可能性のある製品を発見するのに役立ちます。<br/><br/>**使用場所：**<br/> — 製品の詳細<br/> — 買い物かご<br/> — 確認&#x200B;<br/><br/>**推奨ラベル：**<br/> — この究極の購入を見たお客様<br/> — 顧客が最終的に購入した<br/> — 他のユーザーが商品を見た後、何を買うのですか？ |
| これを購入し、それを購入しました | 買い物客が現在閲覧中の製品を使用して過剰に買い物をする製品を推奨します。 買い物かごや製品の詳細ページで最も頻繁に使用されるのは、関連するクロス販売製品の露出を増やして平均注文額を増やすためです。 他の買い物客が現在の商品と共に購入した内容を集計して、関連性の高い商品を買い物かごに追加できるように表示します。<br/><br/>**使用場所：**<br/> — 製品の詳細<br/> — 買い物かご<br/> — 確認&#x200B;<br/><br/>**推奨ラベル：**<br/> — 必要なものをすべて入手<br/> — これらを忘れないでください<br/> — よく一緒に購入 |
| その他の関連ヘルプ | 名前、説明、カテゴリ割り当て、属性など、類似のメタデータに基づいて製品をレコメンドします。 は、表示されている製品の属性を評価することで、同じカテゴリの類似製品を推奨します。 例えば、買い物客がヨガマットを閲覧している場合は、機器カテゴリの他の製品が推奨されます。 このレコメンデーションタイプは性別を区別しないので、衣料品、ファッション、その他の性別固有の業種にはお勧めしません。<br/><br/>**使用場所：**<br/> — 製品の詳細<br/> — 買い物かご<br/> — 確認&#x200B;<br/><br/>**推奨ラベル：**<br/>  — このような他の製品<br/> — 次に類似 |
| [視覚的類似性](#visualsim) | 表示されている製品に類似した商品をレコメンドします。 このレコメンデーションタイプは、商品の画像や視覚的側面が買い物エクスペリエンスにとって重要な場合に最も役立ちます。 |

## 人気度 {#popularity}

これらのレコメンデーションタイプは、過去 7 日間で最も人気のある製品やトレンドを示す製品をレコメンデーションします。

| タイプ | 説明 |
|---|---|
| 最も頻繁に閲覧された | 過去 7 日間に表示アクションが発生したセッション数をカウントすることで、最も多く閲覧された製品をレコメンドします。<br/><br/>**使用場所：**<br/> — ホームページ<br/> — カテゴリ<br/> — 製品の詳細<br/> — 買い物かご<br/> — 確認&#x200B;<br/><br/>**推奨ラベル：**<br/> — 最頻使用<br/> — トレンド<br/> — 現在人気<br/> — 最近人気<br/> — この製品 (PDP) を参考にした人気の製品<br/> — トップセラー |
| 最も多く購入された | 過去 7 日間に買い物客が最も頻繁に購入した製品をレコメンドします。<br/><br/>**使用場所：**<br/> — ホームページ<br/> — カテゴリ<br/> — 製品の詳細<br/> — 買い物かご<br/> — 確認&#x200B;<br/><br/>**推奨ラベル：**<br/>  — 最頻使用<br/> — トレンド<br/> — 現在人気<br/> — 最近人気<br/> — この製品 (PDP) を参考にした人気の製品<br/> — トップセラー |
| 買い物かごに最も多く追加されました | 過去 7 日間に買い物客が買い物かごに最も頻繁に追加した製品をレコメンドします。 このレコメンデーションタイプは、すべてのページで使用できます。<br/><br/>**使用場所：**<br/> — ホームページ<br/> — カテゴリ<br/> — 製品の詳細<br/> — 買い物かご<br/> — 確認&#x200B;<br/><br/>**推奨ラベル：**<br/>  — 最頻使用<br/> — トレンド<br/> — 現在人気<br/> — 最近人気<br/> — この製品 (PDP) を参考にした人気の製品<br/> — トップセラー |
| トレンド | サイト全体での製品の人気の最近の勢いに基づいて、製品をレコメンドします。<br/><br/>Adobe Senseiはサイト全体の閲覧と購入データを集計して、買い物客に最も人気の高い製品を特定し、ランク付けします。 トレンド分析は最近の製品の勢いを分析するので、売上高の高いカタログに対する効果的なレコメンデーションタイプです。 カタログの方が静的な場合は、オーディエンスの購入パターンが非常に変化しない限り、あまり役に立たない可能性があります。<br/><br/>ホームページで使用する場合、トレンド分析では、サイト全体で最近人気があった製品を推奨します。 トレンドは、一貫して人気のある製品ではなく、最近人気が出た製品を表示します。 例えば、特定の製品をプロモーションする電子メールマーケティングキャンペーンがある場合、電子メールによって生成される人気が高まると、プロモーションされた製品がトレンドとして分類される可能性が高くなります。<br/><br/>**使用場所：**<br/> — ホームページ<br/> — カテゴリ<br/> — 製品の詳細<br/> — 買い物かご<br/> — 確認&#x200B;<br/><br/>**推奨ラベル：**<br/> — トレンド<br/> — トレンドを表示<br/> — 最近のトレンド<br/> — 注目の製品<br/> — 関連製品 (PDP) のトレンド分析 |

## 視覚的類似性 {#visualsim}

The _視覚的類似性_ レコメンデーションタイプは、表示されている製品に類似した商品をレコメンデーションすることを推奨します。 このレコメンデーションタイプは、商品の画像や視覚的要素が買い物エクスペリエンスの重要な部分である場合に最も役に立ちます。

### 仕組み

The _視覚的類似性_ レコメンデーションタイプは、現在表示中の画像と視覚的に類似した、カタログ内の他の製品に関するレコメンデーションを提供します。 視覚的類似性には、次のような要素が含まれます。

- カラー
- シェイプ
- サイズ
- テクスチャ
- 素材
- スタイル

Adobe Senseiは AI を使用してカタログ内の画像を処理および分析し、視覚的な類似性を判断するために使用する属性を作成します。

>[!NOTE]
>
> このレコメンデーションタイプを実稼動以外の環境でテストする場合は、画像 URL が公開アクセス可能であることを確認してください。

>[!NOTE]
>
> 現在、製品画像のサイズは 10 MB 以下にする必要があります。

このレコメンデーションタイプはほとんどのカタログには適用されないので、デフォルトでは有効になっていません。 このレコメンデーションタイプを明示的に有効にする必要があります。

### 視覚的類似性レコメンデーションタイプを有効にする

>[!NOTE]
>
> The _視覚的類似性_ レコメンデーションタイプは、 [install](install-configure.md) これはオプションのモジュールです。

1. 次の日： _管理者_ サイドバー、移動 **マーケティング** > _プロモーション_ > **製品Recommendations** 表示する _製品Recommendations_ ダッシュボード。

1. クリック **設定** （歯車アイコン）を使用して、 _設定_ ページに貼り付けます。

1. Adobe Analytics の _Visual Recommendations_ セクション、選択 **Visual Recommendationsを有効にする**.

1. クリック **変更を保存** 完了したら、

   The [新しいレコメンデーションを作成](create.md) ページが表示されました **視覚的類似性** ページタイプが **製品の詳細**.

Visual Recommendations を有効にした後、 Adobe Senseiが画像処理を開始します。 時間はカタログのサイズによって異なります。
